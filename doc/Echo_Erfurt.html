背景

Erfurt（Jenkins node: TestEcho31786_DrZoidberg）是一個多周邊裝置測試環境， 透過 Acroname USB hub 將不同廠牌的 camera / mic / speaker 輪流接上 Duvel (W4) Baseunit，進行 AV 相容性測試。

問題：最初只有一台 Acroname USB hub，既用來切換 ClickShare Button，也用來切換周邊裝置， 兩個功能共用同一個 usb_switcher config 區段，設計上不夠明確，維護困難。

TEnTo Framework 層的改動（PR #3414）

1. config_labels.py — 新增 SWITCHER enum

class SWITCHER(Const):
    BUTTON_SWITCHER     = "usb_switcher"        # 給 button/dongle
    PERIPHERAL_SWITCHER = "peripheral_switcher"  # 給 camera/mic/speaker（新增）

2. configurator_dut.py — he_usb_switcher() 新增 switcher_type 參數

# 修改前：固定讀 "usb_switcher"
def he_usb_switcher(self) -> Optional[IUSBSwitcher]:
    if not self.cut.check_item_present("usb_switcher"): ...

# 修改後：可指定讀哪個 config 區段
def he_usb_switcher(self, switcher_type=SWITCHER.BUTTON_SWITCHER) -> Optional[IUSBSwitcher]:
    if not self.cut.check_item_present(switcher_type): ...

底層實作（ExtronUSBSwitcher / AtenUSBSwitcher / AcronameUsbHub3p）完全相同，差異只在讀哪個 node config 區段，操作哪套物理硬體。

3. CS_TestClasses_dut.py — base class setUpClass 新增初始化

# 舊：只有一個
cls.he_usb_switcher = cls.cut.helpers.he_usb_switcher()

# 新：兩個各自獨立初始化
cls.he_usb_switcher = cls.cut.helpers.he_usb_switcher()
cls.he_usb_peripheral_switcher = (
    cls.cut.helpers.he_usb_switcher(switcher_type=SWITCHER.PERIPHERAL_SWITCHER)
    if cls.check_switcher_usage(SWITCHER.PERIPHERAL_SWITCHER)
    else None
)

4. jenkins_node_labels.py — 新增 USBPeripheralSwitcher Jenkins label

新增 Jenkins node label peripheral_switcher，node 啟動時自動驗證：

  • 對每個 hosts 裡的 baseunit 建立連線
  • 逐一打開 port_peripheral 內的 USB port，確認 baseunit 偵測到裝置上線
  • 驗證每個 port 的 config 都有 camera 與 speakerphone key

只有通過驗證的 node 才會被打上此 label，test job 才能排到該機器。
