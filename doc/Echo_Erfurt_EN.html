Background

Erfurt (Jenkins node: TestEcho31786_DrZoidberg) is a multi-peripheral test environment that connects different brands of camera / mic / speaker to the Duvel (W4) Baseunit via Acroname USB hub for AV compatibility testing.

Problem: Originally only one Acroname USB hub existed, used for both ClickShare Button switching and peripheral device switching. Both functions shared the same usb_switcher config section, which was unclear in design and difficult to maintain.

TEnTo Framework Changes (PR #3414)

1. config_labels.py — Added SWITCHER enum

class SWITCHER(Const):
    BUTTON_SWITCHER     = "usb_switcher"        # for button/dongle
    PERIPHERAL_SWITCHER = "peripheral_switcher"  # for camera/mic/speaker (new)

2. configurator_dut.py — he_usb_switcher() gained switcher_type parameter

# Before: fixed reading "usb_switcher"
def he_usb_switcher(self) -> Optional[IUSBSwitcher]:
    if not self.cut.check_item_present("usb_switcher"): ...

# After: configurable — specify which config section to read
def he_usb_switcher(self, switcher_type=SWITCHER.BUTTON_SWITCHER) -> Optional[IUSBSwitcher]:
    if not self.cut.check_item_present(switcher_type): ...

The underlying implementation (ExtronUSBSwitcher / AtenUSBSwitcher / AcronameUsbHub3p) is identical. The only difference is which node config section is read and therefore which physical hardware is controlled.

3. CS_TestClasses_dut.py — base class setUpClass initializes both switchers

# Old: only one switcher
cls.he_usb_switcher = cls.cut.helpers.he_usb_switcher()

# New: two independently initialized switchers
cls.he_usb_switcher = cls.cut.helpers.he_usb_switcher()
cls.he_usb_peripheral_switcher = (
    cls.cut.helpers.he_usb_switcher(switcher_type=SWITCHER.PERIPHERAL_SWITCHER)
    if cls.check_switcher_usage(SWITCHER.PERIPHERAL_SWITCHER)
    else None
)

4. jenkins_node_labels.py — Added USBPeripheralSwitcher Jenkins label

A new Jenkins node label peripheral_switcher is validated at node startup:

  • Creates connections to each baseunit listed under hosts
  • Enables each USB port in port_peripheral one by one, verifying the baseunit detects the device
  • Validates that each port config contains both camera and speakerphone keys

Only nodes that pass validation receive this label, allowing test jobs to be scheduled on that machine.
